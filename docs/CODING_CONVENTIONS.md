# tinyState コーディング規約

> v0 草案 (2026-06-03)

<hr>

## 1. 定義

tinyStateを利用したフレームワークや、アプリケーションにおいて、クラス
名の命名や、基本となるオブジェクトの定義について述べる。この規則に完全
に従う必要はないものの、コーディングの見通しをよくするためには、共通の
規則、規約にしたがった方がいい。

c++におけるクラス継承における親・祖先・孫・子孫の概念がある一方、
tinyStateでは、生成されたオブジェクトの生成関係を示す、親・祖先・孫・
子孫の概念がある。これらを前者と区別するために実態親、実態祖先、実態孫、
実態子孫と呼ぶことにしよう。

## 2. 命名規則

基本のオブジェクトの命名は、 ([a-z][a-z0-9]*)([A-Z][A-Za-z0-9]*) という形を
とる。

([a-z][a-z0-9]*) の部分は、フレームワークの中の枠組みを表す。たとえば、
データのためのクラス、関数・機能のためのクラス、tinyStateを祖先とする
（orしない）クラスなどである。後半([A-Z][A-Za-z0-9]*)の部分は、そのク
ラスの固有な名前となる。

ある基本のオブジェクトに対し、そのオブジェクトをサポートするオブジェク
ト、特にtinyStateクラスの場合、「co_」という接頭子をつける。ptsSystem
をサポートするクラスは、co_ptsSystem とする。

フレームワークやアプリケーションを代表するクラスはあえて、上記の規約を
踏襲しないこともありえる。たとえば、tinyStateフレームワークでは、
ts... ts2... という命名をとるが、tinyStateクラスだけは違う。

## 3. 基本となるオブジェクト、クラス

      - 元祖クラス
      - 実態元祖クラス

この二つは、フレームワーク、および、アプリケーションの中での円滑な変数、
機能参照のために使われる。実態元祖クラスのpublicには、フレームワークま
たはアプリケーション全体で参照されるグローバルな変数や機能を定義する。
元祖クラスのオブジェクトは、実態元祖クラスのオブジェクトを指し示すポイ
ンタを持っているので、そのポインタを辿ることにより、グローバル機能を参
照することができる。

tinyState フレームワークでは、以下のことが決められており、tinyStateク
ラスの定義、オブジェクトの生成において、自動的に以下が満たされるように
設計されている。

     - あるフレームワークにおいて元祖（クラス）とは、そのフレームワー
       クによって定義されるクラスであって、その親クラスはあったとして
       もそのフレームワーク外のクラスであるクラス。

     - 実態親は、tinyStateのコンストラクタの第一引数parentで指定された
       オブジェクトのことである。

     - このとき、parent で指定されたオブジェクトにとって、コンストラク
       タで生成されたオブジェクトが実態子である。

     - tsApplication はどのtinyStateからparentをたどっていけば、辿り着
       ける、すべての実態祖先にあたる。いわば実態元祖である。

     - tinyState は実態元祖を指し示すポインタapplication をもっている。

     - tsApplication の親parent=thNULL,application=自分自身となっている。

同様に、tinyStateフレームワークおよび、アプリケーションを開発するとき
には、それぞれのフレームワークにとっての元祖クラス、実態元祖（実態元祖
クラス）を定義するべきである。someFramework を開発する場合には、

     - sfwObject (: public tinyState) を元祖クラス
     
     - sfwApplication (: public sfwObject) を実態元祖のクラス
       フレームワーク(ライブラリ)の実態元祖の命名はxxxApplication
       アプリケーションの実態元祖は、xxxMain とするのもよい。
       
     - sfwApplication の実態親はtsApplication

     - sfwObject の実態親は、sfwObject または、tsApplication,
       sfwApplication 以外は、sfwObject を実態親とする。

     - sfwObject は、sfwApp(名前はアプリケーション・フレームワークごと
       に規定) というsfwApplication を指すポインタを持っている。

     - sfwApplication のpublic インスタンスには、someFramework のグロー
       バル変数や関数などを配置するとよい。
